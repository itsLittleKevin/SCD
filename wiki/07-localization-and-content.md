# 本地化体系与内容治理

## 1. 当前状态

项目已从 TS 内嵌词典迁移为 JSON 资源驱动。

当前资源：

1. `src/ui/locales/zh/common.json`
2. `src/ui/locales/en/common.json`

## 2. 工作流建议

1. 新增文案先加 key，不直接写硬编码。
2. 所有语言保持同结构 JSON。
3. 翻译更新走 JSON 批量流程。
4. CI 增加缺失 key 与未使用 key 检查。

## 3. 新语言接入模板

```text
src/ui/locales/<lang>/common.json
```

步骤：

1. 创建新语言 JSON 并复制 key 结构。
2. 在 `i18n.ts` 注册语言。
3. UI 增加语言切换入口。
4. 构建与页面回归检查。

## 4. 内容一致性规范

1. 术语统一：同一概念仅保留一个中文名称。
2. 状态文案统一：避免“状态/扫描状态”等混用。
3. 建议等级统一：高/中/低、可删除/谨慎/不建议。

## 5. 后续目标

1. 清理 `App.tsx` 中剩余硬编码文案。
2. 引入 i18n lint 脚本，自动发现回归。
3. 为产品运营准备可导出的翻译包流程。
