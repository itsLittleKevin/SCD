# PRD：DiskOrg Control Panel（SCD）本地搜索/压缩/查重与风险分析平台（成品对齐版）

## 1. 文档目的

本 PRD 用于对齐当前已落地成品与后续迭代方向，避免“实现已变更而文档仍停留在早期阶段”的偏差。

## 2. 产品定位

DiskOrg Control Panel（SCD）是面向本地文件治理的桌面工具，聚焦四件事：

1. 搜索与筛选：快速定位文件/目录。
2. 去重与风险：识别 99.999% 内容重复、标记异常风险。
3. 压缩与归档：批量压缩/打包，服务于空间回收与归档。
4. 辅助判断：对选中项给出“这是什么”分析与删除建议。

说明：本产品默认是“分析与建议优先”，不是自动清理器。

## 3. 当前版本能力（已实现）

### 3.1 扫描与索引

1. 支持本地路径扫描，结果实时可见。
2. 支持扫描任务启动、状态查询、暂停、取消。
3. 支持任务历史与任务结果回载。
4. 结果持久化使用 SQLite，扫描快照可落盘。

### 3.2 筛选与展示

1. 支持关键词/规则搜索。
2. 支持后缀筛选（包含/排除）。
3. 支持大小区间筛选与排序。
4. 支持文件/目录视图切换。
5. 支持风险类型筛选。

### 3.3 去重能力

1. 采用内容级去重（分层流程，最终完整内容哈希）。
2. 提供高置信重复分组（99.999% 重复）。
3. 支持重复组聚焦与可释放空间估算。

### 3.4 压缩与归档

1. 单项压缩接口：视频/图片/PDF/打包。
2. 批量压缩接口：视频、批量打包。
3. 归档格式支持 zip 与 tar。
4. 视频支持档位预设；图片支持质量与缩放策略；PDF 支持预设与目标体积策略。

### 3.5 智能分析（这是什么）

1. 对当前项或多选项做“这是什么”分析。
2. 返回摘要、删除建议、置信度、影响、理由。
3. 支持配置模型提供方（OpenAI Compatible / Ollama）与检索开关。
4. 支持未配置模型时的安全降级输出。

### 3.6 自动清理策略（Retention）

1. 可配置扫描快照保留天数。
2. 可配置任务保留天数与数量上限策略。
3. 可配置分析缓存保留天数。
4. 支持启动时与任务后触发清理。

### 3.7 国际化（i18n）

1. 已改为 JSON 资源驱动。
2. 当前资源文件：
   1. src/ui/locales/zh/common.json
   2. src/ui/locales/en/common.json
3. 通过统一 t() 获取词条。
4. 仍存在少量页面硬编码文案，需后续继续收敛。

### 3.8 当前 UI 信息架构

1. 顶部：品牌、摘要指标、语言/日期/时间格式控件。
2. 主体三栏：
   1. 左栏：扫描与筛选。
   2. 中栏：索引结果表格。
   3. 右栏：详情与“这是什么”两个同级模块。
3. 右栏目前采用 4/6（详情/这是什么）纵向分配。

## 4. 安全与边界（当前已生效）

### 4.1 接口安全

1. 敏感操作（删除/复制）要求本地可信来源。
2. 敏感操作要求客户端标识与会话 token。
3. 请求体增加大小限制，超限拒绝。

### 4.2 路径安全

1. 删除/复制目标路径必须位于已授权扫描根目录范围内。
2. 越界路径请求直接拒绝。

### 4.3 任务与资源

1. live 扫描任务支持结束后回收，降低常驻内存压力。
2. tar 打包参数加入安全分隔，规避路径被误解析为选项。

## 5. 非功能需求（当前阶段）

1. 稳定优先：任何单文件失败不影响全局任务。
2. 可解释：风险、跳过、建议均有原因输出。
3. 可恢复：任务、结果、设置可持久化。
4. 可维护：i18n 采用资源文件化管理，支持批量翻译。

## 6. 明确不在当前版本范围

1. 自动批量删除与无人值守清理。
2. 系统级实时文件监听（USN/fanotify 全量接入）。
3. 视觉相似图片/音视频语义去重。
4. 深度归档内容索引（多层压缩包全展开）。

## 7. 下一阶段（建议）

### 7.1 i18n 完整收敛

1. 将 App 中 remaining 硬编码文案全部抽 key。
2. 增加缺失 key 检查脚本（CI 校验）。
3. 增加新语言接入模板（例如 ja/common.json）。

### 7.2 可观测性与质量

1. 增加敏感接口安全用例（来源、token、越界路径）。
2. 增加大请求体与异常输入测试。
3. 增加长时扫描与任务回收压力测试。

### 7.3 产品体验

1. 统一术语字典（中文优先语义一致性）。
2. 增强“这是什么”解释结构化程度（证据来源、结论信号分层）。
3. 压缩结果增加更明确的可回收空间与失败分组视图。

## 8. 验收标准（本版）

1. 核心链路可用：扫描 -> 去重/风险 -> 查看详情 -> 分析建议 -> 压缩/归档。
2. 敏感接口具备本地安全校验和路径边界校验。
3. i18n 词典由 JSON 驱动，zh/en 均可正常显示。
4. 关键构建通过：
   1. npm run build
   2. npm run build:ui

## 9. 版本结论

当前版本已从“扫描去重原型”演进为“本地文件治理工具”。
后续应以“安全、可解释、可维护、可本地化扩展”为主线推进，而非盲目扩展功能面。
