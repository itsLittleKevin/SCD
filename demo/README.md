# SCD Demo 控制台

这是用于 VPS 自动演示的本地预演目录。

## 能力

1. 自动场景连播（播放/暂停）。
2. 三键控制（向前 / 播放暂停 / 向后）。
3. 每个场景可执行真实 UI 动作：点击、输入、滚动、切换。
4. 聚焦引导层：非目标区域变暗、目标区域高亮、旁侧说明气泡。

## 本地运行

1. 启动 UI

```bash
npm run dev:ui
```

2. 打开演示控制台

```text
http://127.0.0.1:5173/demo/
```

演示舞台会自动加载：

```text
http://127.0.0.1:5173/?demoSandbox=1
```

该模式为只读沙盒：

1. 不连接真实 API 流程。
2. 不执行删除/复制/压缩等真实操作。
3. "这是什么"返回本地模拟结果，仅用于演示交互。

## 交互说明

1. 页面采用全屏直出，不再保留侧边栏和“演示舞台”容器。
2. 进入页面后自动开始导览。
3. 顶部按钮区提供：
4. “重置导览 / Reset”：可在跳过后一键重新开始。
5. “语言切换”：在简中和 English 间切换，并同步到 Demo 内应用。
6. 每个场景默认停留约 5 秒，也可通过气泡按钮立即切到“上一条/下一条”或“跳过导览”。
7. 说明气泡包含步骤编号，便于现场讲解和节奏控制。

3. 可选：若需要完整 API 行为，另启 API

```bash
npm run dev:api
```

## VPS 部署建议

1. 让主应用与 demo 同域部署，确保 iframe 同源可控。
2. 保留 `/demo/*` 静态文件直出。
3. 将 `/wiki/*` 也一并静态托管。
4. 若启用反向代理，保持 `/api` 可访问（或在 demo 场景中避免依赖 API 路径）。

## 后续可扩展

1. 录制模式：用户手动操作生成场景脚本。
2. 配置模式：场景参数（停留时长、语言、目标关键词）JSON 化。
3. 导出模式：自动演示时同步录屏，产出 mp4/gif。
